swagger configuration:
Swagger configuration means setting up Swagger in your application so it can automatically create API documentation

Swagger configuration tells Swagger:
Project name
API version
Description of the APIs
Which API endpoints to show

Why use Swagger?

✅ Automatically generates API documentation
✅ Easy to test APIs from a browser
✅ Helps frontend and backend developers work together
✅ Saves time writing documentation manually

Why developers use Swagger in FastAPI
See all endpoints in one place.
Test APIs without using Postman.
Understand request and response formats.
Share API documentation with frontend developers.

So when someone says "configure Swagger in FastAPI", they usually mean:

Set API title, version, and description.
Add endpoint descriptions and examples.
Customize the /docs page if needed.

The best part is that FastAPI gives you Swagger UI automatically at /docs.

Dependency::

In FastAPI, a dependency is a way to tell FastAPI:

"Before executing this endpoint, run this function (or class) and provide its result."
It is part of FastAPI's Dependency Injection (DI) system, which helps you reuse common logic without repeating code.

Why use dependencies?

Dependencies are commonly used for:
Authentication and authorization
Database sessions
Validating API keys or tokens
Loading configuration
Logging
Rate limiting
Reusing common validation logic
